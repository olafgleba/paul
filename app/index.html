Paul

Paul is my personal frontend development environment. It relies on libraries and extensions such as Bower, Grunt, Sass and the inuit.css framework as the foundation to build up the real css stuff. Instead to fiddle about repetitive, boring tasks, it allows me to focus on what i do like most: Building websites and apps.

Introduction

Paul allows you to assemble libraries easily, starts a livereload- and webserver in your prefered browser, watch out for changes in your files and folders to run appropriate tasks then, like concatenation, minification, file copy and and a bunch of several more tasks. It also includes a HTML5 boilerplate (template.html) to start with.

Altough it is highly configurable, the main attempt is workflow simplification and staying focused on your real work. Simply run grunt (e.g. grunt-deploy) on the console and forget about the magic that happens behind.

Paul just reflects my way of doing things, my approach to facilitate starting new projects. Hopefully someone find it helpful. By using it as it is or just as a inspiration to build your own environment.

Requirements

You must have installed the latest Node.js, Grunt, Bower and SASS. As currently no windows compatible bash scripts are included, a UNIX operation system (like OS X) is required.

Get Paul

Make a choice:

  • - Clone the git repo: git clone https://github.com/olafgleba/paul.git
  • - Download the zip, extract it and have a go.

Getting started with Paul

  1. Open the Grundfile.js and set your prefered browser (search for $BROWSER)
  2. Execute both build-bower-components.command and build-node-modules.command by double-click or run it from the console (currently only UNIX compatible)
  3. Run $ grunt on the console
  4. You're done. Start development...

Usage of Paul

There are a few rules you have to obey:

  • All of your working files (like CSS, javascript, images etc.) lives within the appropriate source/... folder. Never put any of this stuff directly into the app/ folder.

What Paul does for us

  • Automatically build custom Modernizr tied to what's really been used in your CSS/Javascript files
  • CSS prefixes are automatically added on deployment
  • HTML5 template as a boilerplate to start with

Prepared components/modules

To ensure you always work with the most current component/module version, these files are not included in the project download actually. But installing them is fairly simple: execute both *.command bash scripts that comes with Paul and you're done (see above Get started with Paul).
